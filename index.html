FastAPI + SQLAlchemy Reflection

db2api-service

一个面向关系型数据库的动态 API 服务。它通过反射数据库 schema，自动暴露通用的 CRUD 接口，特别适合自动化测试、测试数据准备、管理后台、内部工具和需要快速验证数据库结构的场景。

A dynamic API service for relational databases. It reflects database schemas and exposes generic CRUD endpoints, making it especially useful for automation testing, test data setup, admin consoles, internal tools, and fast schema validation.

核心能力

Core capabilities

  • 动态表发现 基于 SQLAlchemy 反射读取目标数据库中的表、列和主键信息，无需为每张表手写模型或路由。
  • 通用 CRUD 接口 对每张表统一提供读取、创建、更新、删除能力，适合作为管理后台、内部工具或数据中台的 API 层。
  • 元数据自动刷新 支持定时刷新，也支持通过管理接口立即刷新，方便在 DDL 变化后快速同步新结构。
  • 查询、排序、分页 GET /api/{table} 支持按列名做简单等值过滤，并提供 order_by、desc、limit、offset 参数。
  • 适合自动化场景 可以直接作为自动化测试的数据访问层，用于接口联调、回归测试、测试环境数据准备，以及快速搭建面向数据库表的内部服务。
  • Dynamic table discovery Uses SQLAlchemy reflection to read tables, columns, and primary keys without hand-writing a model or route for every table.
  • Generic CRUD endpoints Exposes consistent create, read, update, and delete operations per table, which fits internal tools, admin consoles, and data platforms.
  • Metadata refresh Supports both scheduled refresh and manual refresh endpoints so schema changes can be picked up immediately after DDL updates.
  • Filtering, sorting, and paging GET /api/{table} supports simple equality filters by column name plus order_by, desc, limit, and offset.
  • Fits automation workflows Works well as a data-access layer for automated testing, API integration checks, regression suites, test environment data setup, and quickly exposing internal database-backed services.

接口示例

Endpoint examples

  • GET /metadata 查看当前反射到的表、列和数据库连接摘要。
  • POST /admin/refresh 强制刷新元数据，立即重新读取数据库 schema。
  • GET /api/users?name=Alice&order_by=created_at&desc=true&limit=20&offset=40 使用等值过滤、单字段排序和分页读取列表数据。
  • PATCH /api/users/1 以主键定位单条记录，并更新指定字段。
  • GET /metadata Inspect reflected tables, columns, and the active database connection summary.
  • POST /admin/refresh Force a metadata refresh and immediately reload the database schema.
  • GET /api/users?name=Alice&order_by=created_at&desc=true&limit=20&offset=40 Read list data with equality filtering, single-column sorting, and pagination.
  • PATCH /api/users/1 Update a single row by primary key.

典型用途

Typical uses

  • 自动化测试与回归验证 测试脚本可以直接调用通用接口准备数据、校验写入结果，并在回归执行后清理测试记录。
  • 测试环境数据准备 在没有专门后端接口的情况下，快速为测试环境插入、查询和修正表数据。
  • 内部管理与运维工具 为后台运营、客服排障、数据核查这类内部场景快速提供可访问的表级 API。
  • 数据库结构快速验证 当数据库 schema 频繁变化时，可以用统一接口快速验证列变化、主键行为和基础 CRUD 是否仍然正常。
  • Automation testing and regression validation Test suites can call the generic endpoints to seed data, validate writes, and clean up records after execution.
  • Test environment data setup Quickly insert, query, and fix table data even when no dedicated backend endpoint exists yet.
  • Internal admin and operations tools Expose table-level APIs for support teams, operators, and data-check workflows without building a full custom backend first.
  • Fast schema verification When schemas change often, use the same API layer to verify column changes, primary key behavior, and baseline CRUD still work.
